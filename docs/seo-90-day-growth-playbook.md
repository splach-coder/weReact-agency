# WeReact 90-Day SEO Growth Playbook

## Success Metric

Measure qualified project enquiries, not rankings alone: submitted contact forms, WhatsApp conversations, booked intro calls, and the source/landing page that produced each one.

## Week 1: Make the new site visible

1. Deploy the current `master` branch so production no longer exposes stale About and Services navigation.
2. In Google Search Console, add both `https://www.wereact.agency` and the `www` canonical property if needed, submit `https://www.wereact.agency/sitemap.xml`, then inspect and request indexing for the home page, three service pages, work page, contact page, and the three articles.
3. Add the same sitemap to Bing Webmaster Tools.
4. Use Google Tag Assistant and GA4 DebugView to submit one test contact form and confirm `generate_lead` plus the Ads conversion event fire only after consent is granted.
5. Test each new page on a real phone: headline visible, WhatsApp opens, contact CTA works, no horizontal scrolling, and text remains readable.

## Weeks 2-4: Establish local entity trust

1. Complete and verify the Google Business Profile using the same business name, category, phone, email, website, city/service area, hours, logo, and real photos shown on the website.
2. Add services to the profile using the same wording as the service pages. Avoid invented pricing, awards, reviews, or performance claims.
3. Request a short, honest Google review from each client only after delivery. Never offer incentives or write a review for a client.
4. Build only accurate citations: local business directories, relevant Moroccan business associations, partner/travel listings, and real client website credits. Keep name, phone, and website URL identical everywhere.
5. Ask each approved client for one contextual backlink, such as a portfolio credit or a project announcement linking to the matching WeReact case study.

## Weeks 5-8: Publish the first topic cluster

Publish one practical article every two weeks. Each article needs a named WeReact byline, first-hand project context, one service-page link, one case-study link, one image with descriptive alt text, and a single project CTA.

1. `Combien coûte la création d'un site internet à Marrakech ?` Explain scope drivers rather than publish fake fixed prices.
2. `Site web pour riad à Marrakech : pages, photos, réservation et demandes directes.`
3. `Google Ads au Maroc : pourquoi une landing page dédiée convertit mieux qu'une page d'accueil.`
4. `Site bilingue français-anglais : structure SEO pour entreprises marocaines.`

Do not produce city pages that merely swap Marrakech for Casablanca, Rabat, or Agadir. Publish a location page only when WeReact has genuine local work, local context, and useful distinct guidance for that city.

## Weeks 9-12: Improve based on evidence

1. In Search Console, review impressions, click-through rate, queries, indexed pages, and page-level performance. Improve titles/descriptions on pages with impressions but weak CTR.
2. In GA4 and Google Ads, compare qualified leads by landing page, campaign, device, and first source. Keep ads pointed to the dedicated landing page that matches the keyword intent.
3. Review form submissions and WhatsApp messages. Add FAQs only where prospects repeatedly ask the same question.
4. Update one existing article or case study each month with real details, newly approved proof, and a fresh `modifiedAt` date.

## Reporting Template

Track monthly:

- Organic impressions, clicks, CTR, and average position by page and query.
- Indexed service/article URLs and crawl errors.
- Contact forms, WhatsApp clicks, booked calls, and qualified leads.
- Google Ads cost per qualified lead, not just clicks.
- New verified reviews, citations, and client backlinks.
- Mobile usability and Core Web Vitals warnings in Search Console.

## Guardrails

- Rankings take time; no agency can guarantee a position or a volume of leads.
- Keep conversion tracking consent-aware.
- Use only client names, project claims, logos, and quotes that are approved and verifiable.
- Remove or revise stale pages instead of allowing old navigation and old claims to remain indexed.
